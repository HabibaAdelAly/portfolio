Networking

The networking layer was designed using a Hub-and-Spoke architecture to provide centralized management, secure communication, and scalable hybrid connectivity between Microsoft Azure and the on-premises infrastructure.

Overview

The Azure environment separates infrastructure services from production workloads using dedicated Virtual Networks and subnets. Shared services such as Azure Firewall, Azure Bastion, VPN Gateway, monitoring, and DNS are hosted inside the Hub Virtual Network, while application resources are deployed within isolated spoke networks. This design improves scalability, simplifies management, and minimizes lateral movement between workloads.

Architecture

The primary region contains a Hub Virtual Network hosting shared networking services including Azure Firewall, Bastion, VPN Gateway, monitoring resources, and management services. Production workloads are deployed within a dedicated Spoke Virtual Network containing separate Application and Data subnets. Communication between VNets is achieved through VNet Peering while maintaining network isolation.

Hybrid Connectivity

  • ExpressRoute provides the primary private connection between Azure and the on-premises environment.
  • Site-to-Site VPN acts as a redundant backup connection during outages.
  • VPN Gateway terminates encrypted tunnels before routing traffic into Azure.
  • Traffic remains on Microsoft's backbone network whenever possible.
  • Private Endpoints provide secure access to Azure PaaS services.

Network Segmentation

  • Dedicated AzureFirewallSubnet for centralized traffic inspection.
  • GatewaySubnet hosting VPN Gateway resources.
  • AzureBastionSubnet providing secure remote administration.
  • ManagementSubnet for administration services.
  • SharedServicesSubnet hosting DNS and monitoring resources.
  • Application Subnet isolated from Data Subnet.
  • Private communication between workloads using VNet Peering.

Traffic Flow

External traffic enters through Azure Firewall where security policies are evaluated before requests are forwarded toward application workloads. Application traffic is distributed through an Internal Load Balancer while database communication remains private using Private Endpoints. Hybrid connectivity enables secure communication between Azure resources and on-premises Active Directory, DNS, and file services.

Key Design Decisions

  • Centralized Hub-and-Spoke network topology.
  • Dedicated subnets for infrastructure and workloads.
  • Layered hybrid connectivity using ExpressRoute and VPN.
  • Private service access through Private Endpoints.
  • High availability using Global VNet Peering.
  • Scalable architecture supporting additional spokes.
  • Reduced attack surface by limiting public exposure.
  • Enterprise-ready network segmentation.
